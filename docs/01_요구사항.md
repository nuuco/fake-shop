# Tomato Market · 요구사항

> 과제 6 React 쇼핑몰.  
> 구현된 앱 기준. 설계·AI 과정은 [02](./02_데이터구조.md)·[PROMPT_LOG](./PROMPT_LOG.md)·[07](./07_과제제출체크.md) 참고.

## 1. 앱 한 줄 소개

게스트·로그인 사용자가 Fake Store 상품을 조회하고, Redux 전역 장바구니에 담아 총액을 확인하며, Firebase로 로그인·로그아웃하는 React 쇼핑몰입니다.

## 2. 요구사항 정의표

| 항목 | 내용 |
|---|---|
| 앱 이름 | Tomato Market (레포 `fake-shop`) |
| 한 줄 설명 | 상품 조회 · 전역 cart · Firebase 인증 |
| 기술 | React, TypeScript, Redux Toolkit, Firebase Auth, React Router, Vite |
| 상품 데이터 | Fake Store API 우선, 실패·timeout 시 mock |
| 장바구니 저장 | Redux(메모리) + LocalStorage |
| 인증 | Google popup + 이메일/비밀번호 |
| 필수 | 목록·담기·총액·로그인/로그아웃·로딩/에러/빈 |
| 권장 | 수량·삭제·재시도·인증 로딩 UX·상세 |
| 도전 | TS·검색·카테고리·LocalStorage·배지 |
| 제외 | 결제·주문·배송·회원 등급·실제 개인정보 DB 저장 |

## 3. 사용자 흐름

```text
탐색(목록·검색·필터·상세)
→ 담기(헤더 미리보기·배지)
→ (선택) 로그인
→ 장바구니(수량·총액·계산하기)
→ 로그아웃( cart 유지 )
```

## 4. 화면 ↔ 기능 매핑

| 화면 | 경로 | 기능 |
|---|---|---|
| 홈 | `/` | 상품 목록, loading/error/empty/mock, 검색·필터, 담기 |
| 상세 | `/product/:id` | 상세 조회, 담기 문구·장바구니 이동 |
| 장바구니 | `/cart` | 목록·수량·삭제·총액·계산하기(clearCart)·빈 안내 |
| 로그인 | `/login` | Google·이메일 로그인/가입, 초기 로딩·오류 |
| Header | 공통 | 배지, 담기 미리보기, Login/Logout, 인증 로딩 |

## 5. API·예외 상태

| 상태 | 화면 처리 | 확인 |
|---|---|---|
| loading | 「상품을 불러오는 중입니다…」 | 최초·재시도 (`HomePage`) |
| success | ProductList 그리드 | Fake Store 목록 |
| error + mock | 경고 배너 + MOCK_PRODUCTS + 다시 시도 | timeout/네트워크 |
| empty (목록 자체 0건) | 「표시할 상품이 없습니다.」 | API/mock 결과가 빈 배열 |
| empty (검색·필터 0건) | 「조건에 맞는 상품이 없습니다.」 | `ProductList` |
| auth loading | Login: 「인증 상태를 확인하는 중입니다…」 / Header: `…` | 새로고침 직후 |
