## 1. API timeout · mock 대체

### 증상
상품 로딩이 길게 이어지거나 실패해 목록이 비는 것처럼 보임.

### 원인
Fake Store API 지연·차단·네트워크 오류. 앱이 무한 대기하면 “멈춘 것”처럼 보임.

### 대안

| 안 | 결과 |
|---|---|
| 실패 시 빈 화면만 | ❌ 과제(앱 미정지) 미충족 |
| 즉시 mock만 사용 | ➖ API 실연동 약함 |
| API 우선 + timeout 후 mock + 재시도 | ✅ 채택 |

### 해결
`productApi.ts`에서 8초 `AbortController` timeout. 실패 시 `MOCK_PRODUCTS` + 오류 메시지. UI에 다시 시도.

### 검증
느린 네트워크/차단 시 mock 배너와 목록 표시. `npm run build` 성공.

### 관련 코드
`src/services/productApi.ts`, `src/services/mockProducts.ts`, `src/pages/HomePage.tsx`

## 2. RootState 순환 import

### 증상
(설계·빌드) `cartSlice`와 `store`가 서로를 import하면 타입 순환.

### 원인
selector 타입에 `RootState`를 store에서 가져오고, store는 reducer를 slice에서 가져옴.

### 대안

| 안 | 결과 |
|---|---|
| 공통 types 파일로 RootState 분리 | ➖ 가능하나 과함 |
| slice 내부 `CartRootState` 로컬 타입 | ✅ 채택 |

### 해결
`cartSlice.ts`에 `CartRootState = { cart: CartState }` 정의 후 selector에 사용.

### 검증
`npm run build` 성공.

### 관련 코드
`src/store/cartSlice.ts`

## 3. 사이드이펙트 리팩토링

### 증상 (예방·수정)
- StrictMode에서 fetch 레이스
- LocalStorage 예외 시 subscribe 깨질 위험
- auth `error`인데 user가 남는 등 UI 모순

### 원인
unmount 시 abort 없음, 매 dispatch마다 persist, auth status를 액션에서 직접 덮음.

### 대안

| 안 | 결과 |
|---|---|
| 현상 유지 | ❌ |
| Abort + persist 가드 + listener 전담 status | ✅ 채택 |

### 해결
1. Home/Detail fetch에 AbortSignal, cleanup에서 abort  
2. cart 참조 변경 시에만 `persistCart` + try/catch  
3. `onAuthStateChanged`만 `status`/`user` 갱신

### 검증
목록·담기·로그인 UI가 새로고침·StrictMode에서도 일관.

### 관련 코드
`src/pages/HomePage.tsx`, `src/pages/ProductDetailPage.tsx`, `src/store/index.ts`, `src/hooks/useAuth.ts`

## 4. Firebase popup COOP 경고

### 증상
콘솔에 `Cross-Origin-Opener-Policy` / `window.closed` 관련 메시지 반복.

### 원인
`signInWithPopup`과 브라우저 COOP 정책. Firebase 쪽 흔한 경고.

### 대안

| 안 | 결과 |
|---|---|
| `signInWithRedirect`로 전환 | ➖ 가능, UX 변경 |
| 치명 오류 아님으로 판정, Header로 성공 확인 | ✅ 채택 |

### 해결
로그인 성공 여부는 Header 이니셜·Logout 표시로 확인. popup 유지.

### 검증
실 Google 로그인 후 헤더 상태 확인.

### 관련 코드
`src/hooks/useAuth.ts`, [06](./06_Firebase설계.md)

## 5. Vercel SPA 새로고침 404

### 증상
배포 후 `/cart`, `/login`에서 새로고침 또는 URL 직접 입력 시 `404 NOT_FOUND`.

### 원인
React Router 클라이언트 경로인데, Vercel이 해당 경로의 정적 파일을 찾음.

### 대안

| 안 | 결과 |
|---|---|
| HashRouter로 변경 | ➖ URL 품질↓ |
| `vercel.json` rewrite → `index.html` | ✅ 채택 |

### 해결

```json
{
  "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }]
}
```

재배포.

### 검증
`https://tomato-market-one.vercel.app/cart` 새로고침 정상.

### 관련 코드
`vercel.json` · [08](./08_데모접속_사이트URL.md)
