# Tomato Market · 데이터 구조

> 사람이 먼저 `product` / `cartItem` / `authUser`를 설계한 뒤 AI에 기능 단위로 구현을 맡겼습니다.  
> 관련: [01](./01_요구사항.md) · [06](./06_Firebase설계.md) · 코드 `src/types/*`, `src/store/cartSlice.ts`

## 1. 상태 경계 (의도적 분리)

| 영역 | 위치 | 이유 |
|---|---|---|
| cart | Redux Toolkit (`configureStore` + `createSlice`) | 과제 필수·전역 UI 상태 |
| auth | React Context + Firebase (`useAuth`) | Firebase listener 중심, Redux 과다 사용 회피 |
| product 목록 | 페이지 로컬 state + `productApi` | 서버 데이터, 전역 불필요 |
| cart 영속 | LocalStorage (`fake-shop-cart`) | 새로고침 유지(도전). Firestore 실동기화는 **설계만** |

## 2. `product`

| 필드 | 자료형 | 원본 | 사용 |
|---|---|---|---|
| id | number | API id | key·상세 라우트 |
| title | string | title | 카드·cart |
| price | number | price | 단가·총액 |
| image | string | image | 카드·상세 |
| category | string | category | 필터 |
| description | string | description | 상세 |

- 변환: `mapApiProduct` (`src/services/productApi.ts`)
- 실패·8초 timeout → `MOCK_PRODUCTS` (동일 스키마)

## 3. `cartItem`

| 필드 | 자료형 | 출처 | 규칙 |
|---|---|---|---|
| productId | number | product.id | 불변 |
| title | string | product.title | 불변 |
| price | number | product.price | 계산용 |
| quantity | number | 기본 1 | ≥1, decrease 시 1 이하면 삭제 |
| image | string | product.image | 표시용 |

### action · selector

| 구분 | 이름 | 역할 |
|---|---|---|
| action | `addItem` | 담기, 중복 시 quantity+1 |
| action | `increase` / `decrease` | 수량 |
| action | `removeItem` | 항목 삭제 |
| action | `clearCart` | 계산하기 |
| action | `clearLastAdded` | 헤더 미리보기 해제 |
| selector | `selectCartItems` | 목록 |
| selector | `selectCartTotal` | **Σ(price × quantity)** — state에 total 미저장 |
| selector | `selectCartCount` | Header 배지 |
| selector | `selectLastAdded` | 담기 미리보기 |
| selector | `selectIsInCart` | 상품이 cart에 있는지 (상세는 인라인 `.some`도 사용) |

### 정책

| 항목 | 선택 |
|---|---|
| 같은 상품 재추가 | quantity + 1 |
| 최소 수량 | 1 |
| 로그아웃 시 cart | **유지** (게스트 허용) |
| 저장 | Redux + LocalStorage |

## 4. `authUser`

| 필드 | 화면 | 보호 |
|---|---|---|
| uid | 미노출 | 캡처·UI에 표시 안 함 |
| displayName | 이니셜·툴팁 | 전체 이름은 aria/툴팁만 |
| email | displayName 없을 때 마스킹 | 실이메일 전체 미노출 |
| photoURL | 미사용 | |

`AuthStatus`: `loading` | `authenticated` | `unauthenticated` | `error`

> Footer의 주소·상호는 **브랜드 데모 문구**이며 실사용자 개인정보가 아님.

## 5. 사용자별 cart 클라우드 (설계만 · 미구현)

```text
경로: users/{uid}/cart/items/{productId}
필드: productId, title, price, quantity, updatedAt
규칙 초안: request.auth != null && request.auth.uid == userId
```

현재는 LocalStorage만 사용. 개인정보·주문 실저장 금지 범위 준수.
